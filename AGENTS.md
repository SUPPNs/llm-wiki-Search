# Research LLM Wiki - 編集規約

このリポジトリは、論文研究のテーマ探索に使う知識ベースである。LLM は原資料を
保存する担当ではなく、原資料から追跡可能な知識ページを育てる担当として振る舞う。
Obsidian で人間が読み進めやすい日本語の見出しと表示名を優先する。

## 基本レイヤー

```text
raw/       変更禁止: 人間または収集工程が配置した原資料
wiki/      LLM編集可: 整理、比較、研究テーマ、問いの知識ページ
logs/      追記専用: 重要な wiki 更新の記録
scripts/   任意: 検査・補助スクリプト
```

## 必須ルール

1. `raw/` は変更禁止である。LLM は `raw/` 以下のファイルを作成、編集、移動、
   改名、削除してはならない。読み取りと `情報源` での参照のみ許可する。
2. `wiki/` は LLM が編集してよい。編集前に `wiki/index.md` と関連する既存
   ページを読み、新規作成より既存ページの更新を優先する。
3. 新規作成または内容更新する知識ページでは、`事実`、`解釈`、`未解決課題` を
   明確に分ける。資料が直接裏付けない主張を `事実` に書いてはならない。
4. 研究テーマ候補は `wiki/themes/` に保存する。
5. 未解決課題は `wiki/questions/` に保存する。
5a. 原資料がシステム構成や運用フローを主要成果として提示する場合、
   アーキテクチャ整理ページは `wiki/architectures/` に保存してよい。
6. `wiki/` 配下の各知識ページには `## 情報源` セクションを必ず置き、
   根拠となる `raw/` のパス、URL、または他の根拠ページを列挙する。
7. 既存の記述と新しい根拠が矛盾する場合、古い記述を黙って削除しない。
   `## 矛盾` に相違、根拠、解消に必要な確認事項を記録する。
8. 重要な追加、統合、矛盾の発見、テーマ候補の状態変更は `logs/log.md` に
   追記する。既存ログは書き換えない。
9. 関連ページへの参照は Obsidian 形式の `[[Wikiリンク]]` を使う。ファイル名は
   英数字のまま維持し、表示名には日本語を用いてよい。
10. 確認できない内容は断定せず、`解釈` または `未解決課題` に置き、
    必要な根拠を示す。
11. 既存ページが英語見出しを使用している場合、内容を次に更新する際に、
    本規約の日本語見出しへ合わせて移行する。

## フォルダ構成

| フォルダ                | 表示名       | 内容                        |
| ------------------- | --------- | ------------------------- |
| `raw/papers/`       | 原資料 / 論文  | 取得した PDF や論文資料。変更禁止       |
| `raw/web/`          | 原資料 / Web | 保存した Web 資料。変更禁止          |
| `raw/youtube/`      | 原資料 / 動画  | 字幕や動画由来資料。変更禁止            |
| `raw/notes/`        | 原資料 / メモ  | 人間が保存した一次メモ。変更禁止          |
| `wiki/papers/`      | 論文        | 論文ごとの問題設定、方法、結果、限界、研究上の接続 |
| `wiki/concepts/`    | 概念        | 概念、手法、データセット、指標、理論的枠組み    |
| `wiki/architectures/` | アーキテクチャ | システム構成、要素間の役割、運用・開発フロー       |
| `wiki/themes/`      | 研究テーマ候補   | 実行可能性を検討する研究テーマ候補         |
| `wiki/questions/`   | 未解決課題     | 矛盾や追加検証が必要な問い             |
| `wiki/comparisons/` | 比較        | 複数の方法、論文、設定を同一軸で比較する表     |
| `wiki/templates/`   | テンプレート    | 新しい知識ページを作るときの雛形          |

## ページの必須構成

知識ページには、ページ種別に応じたテンプレートを使う。最低限、次の日本語
セクションを保つ。

| セクション | 内容 |
| --- | --- |
| `概要` | ページの対象と研究探索上の意味の短い説明 |
| `事実` | 情報源から直接支持できる記述。項目ごとに根拠を対応付ける |
| `解釈` | 根拠に基づく整理、含意、仮説。事実として扱わない |
| `未解決課題` | 調査や実験で確かめる必要がある問い。可能なら `wiki/questions/` に接続 |
| `関連ページ` | `[[Wikiリンク]]` による論文、概念、テーマ、比較への接続 |
| `矛盾` | 食い違う主張や証拠。なければ `- 現時点で確認された矛盾はない。` を残す |
| `情報源` | 原資料の場所と、その資料が支える範囲 |

## ファイル名と Wiki リンク

- 新規ファイル名は原則として英数字とハイフンによる `kebab-case.md` を使う。
- フォルダ名と既存ファイル名は変更しない。
- Obsidian 上の表示名は日本語を許可し、たとえば
  `[[themes/efficient-evaluation|効率的な評価手法の研究]]` の形式にする。
- ページのタイトルや用語を変更する場合は、参照元ページのリンクも確認する。
- 同義語で別ページを増やさない。既存ページに別名や用語差分を記録する。

## 原資料を取り込む手順

1. `AGENTS.md`、`wiki/index.md`、関連しそうな既存ページを読む。
2. 指定された `raw/` 資料を読み、引用可能な事実、LLM の解釈、残る問いを分ける。
3. 該当する既存の論文・概念・アーキテクチャ・研究テーマ候補・未解決課題・比較ページを更新する。
   該当ページがない場合のみ `wiki/templates/` のテンプレートを基に新規作成する。
4. 新しい関連を `関連ページ` の `[[Wikiリンク]]` で追加する。
5. 異なる主張が見つかった場合は、関係するページの `矛盾` に記録する。
6. 新規ページまたは大きく変えたページを `wiki/index.md` に登録する。
7. 重要な知識更新を `logs/log.md` に日付付きで追記する。

## 研究テーマ候補を育てる手順

1. `wiki/questions/` の未解決課題と `wiki/comparisons/` の差分を確認する。
2. 新しいテーマ候補は `wiki/themes/` に作成し、対象問題、仮説、貢献候補、
   検証方法、必要資源、失敗条件を記す。
3. テーマ候補は必ず根拠となる論文・概念・問いに `[[Wikiリンク]]` で接続する。
4. 実現性や新規性がまだ検証されていない内容は `解釈` または
   `未解決課題` として扱う。

## 更新後の確認

更新後に可能な範囲で次を確認する。

- 新しい知識ページが `wiki/index.md` に載っているか。
- 新規作成または更新したページに `情報源`、`関連ページ`、`矛盾` があるか。
- `事実` と `解釈` が混同されていないか。
- 追加した `[[Wikiリンク]]` の対象ページが存在するか、今後作成する問いとして
  明示されているか。
- 重要な変更が `logs/log.md` に記録されているか。

## テンプレート

- 論文ページ: `wiki/templates/paper.md`
- 概念ページ: `wiki/templates/concept.md`
- アーキテクチャページ: `wiki/templates/architecture.md`
- 研究テーマ候補ページ: `wiki/templates/theme.md`
- 未解決課題ページ: `wiki/templates/question.md`
- 比較ページ: `wiki/templates/comparison.md`
