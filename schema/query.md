# Query 手順

問い合せによって得られた有用な分析を、会話だけで失わず `wiki/` に蓄積するための
規則である。

## 保存対象

- 複数資料・方式の比較として再利用できる分析
- 既存ページ間に見つかった重要な接続
- 検証可能な未解決課題
- 原資料に裏付けられた研究テーマ候補
- 既存知識と新しい根拠の矛盾

## 手順

1. 問いに関連する `wiki/index.md` と既存ページを確認する。
2. 必要な場合のみ `raw/` の情報源を読み、回答の根拠を確保する。
3. 既存ページへの追記で十分か、新しい比較・問い・テーマページが必要かを判断する。
4. 保存する場合は [naming.md](naming.md) と `schema/templates/` に従う。
5. 保存内容を `wiki/index.md` に反映し、`logs/query.md` に追記する。
6. 新しい Wiki リンクについて [lint.md](lint.md) の点検を行う。

## 注意

- 会話での推論のみを根拠に `事実` を増やさない。
- 有用な分析を保存する際も、根拠と解釈を分ける。
- 保存しない一時的な回答は `logs/query.md` に記録する必要はない。
