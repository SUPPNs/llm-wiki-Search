# Wiki Lint 手順

wiki の読みやすさと追跡可能性を定期的に確認するための手順である。

## 点検項目

1. `wiki/index.md` に新規または重要更新ページが登録されているか。
2. 知識ページに `概要`、`事実`、`解釈`、`未解決課題`、`関連ページ`、
   `矛盾`、`情報源` があるか。
3. `[[Wikiリンク]]` の参照先ページが存在するか。
4. 索引以外のページから参照されない孤立ページがないか。
5. `事実` に情報源の裏付けがなく、解釈が混入していないか。
6. 矛盾する主張が削除されず `矛盾` に記録されているか。
7. `raw/` の資料が変更されていないか。
8. `schema/` と `logs/` への参照が現在の構成と一致しているか。

## 実行記録

- lint を実行した日付、対象範囲、検出事項、修正内容を `logs/lint.md` に追記する。
- 問題がなかった場合も、点検範囲と「問題なし」を短く記録してよい。
- 点検によって内容上の修正が必要になった場合は、根拠を確認した上で対象ページと
  適切な ingest または query ログも更新する。
