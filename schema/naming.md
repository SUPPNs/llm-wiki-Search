# 命名規則

この文書は、Research LLM Wiki のファイル名、表示名、リンク、ページ種別の規則を
定める。詳細な編集原則は [AGENTS.md](AGENTS.md) を優先する。

## ディレクトリ

| 場所 | 用途 |
| --- | --- |
| `raw/<source-type>/` | 不変の原資料 |
| `wiki/papers/` | 論文・資料単位の読解 |
| `wiki/concepts/` | 概念、手法、評価指標 |
| `wiki/architectures/` | システム構成と運用フロー |
| `wiki/comparisons/` | 同一軸での比較 |
| `wiki/questions/` | 未解決課題 |
| `wiki/themes/` | 研究テーマ候補 |
| `schema/templates/` | ページ雛形 |

## ファイル名

- 新規 wiki ページは原則として ASCII の `kebab-case.md` を使う。
- 既存の英数字ファイル名は、リンク互換性のため不要に改名しない。
- 原資料のファイル名とパスは変更しない。
- ページ種別は frontmatter の `type` で明記する。

## 表示名とリンク

- ページタイトルと Wiki リンクの表示名には日本語を使用してよい。
- wiki 内のページ参照は `[[concepts/example|日本語表示名]]` 形式を使う。
- `schema/`、`logs/`、`raw/` など wiki 外の文書や資料への参照は通常の
  Markdown リンクまたはコード表記のパスを使う。
- 同義のページを増やさず、既存ページに `aliases` または用語差を記録する。

## 必須見出し

知識ページには `概要`、`事実`、`解釈`、`未解決課題`、`関連ページ`、`矛盾`、
`情報源` を置く。資料が直接述べていない内容は `事実` に含めない。
