# 説得力の高い RAG 推薦を忠実性と公平性を含めてどう評価するか

## 概要

グラフ探索で推薦理由がわかりやすく納得しやすくなったとき、その理由が取得証拠に
忠実で、人気商品や活発なレビューアーへの偏りを過度に増幅していないことをどの
ように同時評価するかを問う。

## 重要性

説得力だけを最適化すると、根拠が弱いが魅力的な説明や、接続数の多い商品に偏った
推薦を評価してしまう可能性がある。研究テーマとして扱うには利用者評価と根拠品質
を同じ実験に含める必要がある。

## 事実

- Arakaki・Fushimi (2026) は、VR+BGR が VR のみより推薦理由のわかりやすさ、納得感、参考度で多く選ばれたと報告する。表4-6、p.14。
- 同論文の BGR は、取得レビューの隣接アイテムとレビューアーのレビュー履歴を追加して推薦理由を生成する。英文概要、3 節。
- 同論文は、コスメに非常に詳しい回答者で提案方式への評価が相対的に低い可能性を述べる。6.5 節、p.15。
- Anaguchi et al. (2025) は、LLM を用いた知識グラフ構築に情報欠損の可能性があると報告する。3.6 節、p.4。

## 解釈

- 推薦理由評価には、利用者が感じる説得力だけでなく、引用レビューとの整合性、主張された特徴の根拠回収率、人気・活動量による露出偏りを加えるべきである。
- 専門知識の高い利用者への説明では、レビュー上の支持に加え、成分や機能の根拠が必要になる可能性がある。

## 解決条件

| 項目           | 内容                                                       |
| ------------ | -------------------------------------------------------- |
| 必要な証拠        | 同一推薦出力に対する説得力評価、根拠忠実性評価、露出偏り分析                           |
| 候補となる実験 / 調査 | VR、VR+BGR、BGR+知識証拠検索を比較するユーザー評価と根拠監査                     |
| 関連する評価指標     | わかりやすさ、納得感、参考度、citation/evidence precision、coverage、露出分布 |
| 解決とみなす条件     | 説得力改善が根拠忠実性を損なわず、偏りの条件と緩和策が説明できる                         |
