# HITL ML の運用データと再学習をどう評価するか

## 概要

人間による修正を運用中に蓄積する HITL ML において、どの量・品質・条件の修正
データが再学習によるモデル改善につながるか、また業務効果とモデル効果をどう
分離して評価するかという問いである。

## 重要性

本論文ではシステム導入による作業負荷削減は報告されたが、運用中の修正データを
用いた再学習による精度改善は確認されなかった。HITL の評価では、運用価値を保ち
つつ、学習改善が成立する条件を明確にする必要がある。

## 事実

- AI 腐食診断システムは、保守員の修正データを再学習へ接続する構成を含む。図7-8、pp.28-30。
- 開発時には、7 人の保守員が炭素鋼の画像 `3,800` 枚をアノテーションした。p.32。
- 判定 B と C の境界にはばらつきがあり、評価基準の `100%` の標準化は実現できなかったと報告される。p.32。
- 運用中に得た診断不一致 `110` 件で再学習した結果、再学習前に比べてモデル精度は改善しなかった。4.3.1 節、p.33。
- 開発時の理想的な画像と運用時の画像に画質差があり、著者らはこの差が再学習時の精度向上を妨げる要因になっていると述べる。4.3.3 節、p.34。
- 保守員は塗装剥離と減肉剥離を区別できたが、モデルは区別できなかったと報告される。4.3.2 節、p.33。

## 解釈

- 本資料は、運用中に人間の修正を得られることと、そのデータで直ちに精度改善できることが同じではないことを示す。
- 追加の原因仮説や解決手法はこのページでは断定せず、論文が指摘したデータ量、ラベル一貫性、画像品質、識別クラスの問題を検証項目として保持する。

## 解決条件

| 項目 | 内容 |
| --- | --- |
| 必要な証拠 | 修正データ量、ラベル一致度、画像品質、再学習前後の同一条件評価、作業負荷の測定 |
| 候補となる実験 / 調査 | 論文が今後の方向として述べる、評価基準の再確認、複数保守員によるアノテーション確認、肉厚測定等の実データによる再アノテーション、クラス再定義の検討 |
| 関連する評価指標 | F 値、Recall、Precision、アノテーション一致度、作業量、再学習前後差 |
| 解決とみなす条件 | 修正データを用いた再学習の改善有無と、改善が成立するデータ・運用条件が実測される |

## 未解決課題

- 再学習の効果が確認できるまでに必要な修正データ量はどの程度か。
- アノテーション基準の経時的なばらつきをどのように測定し管理するか。
- 実運用画像の品質差を抑える撮影方法の改善は、モデル評価にどの程度影響するか。
- 腐食クラスの再定義や剥離種別の区別は、モデル精度と保安上の判断にどのように影響するか。
- HITL の開発・運用連携を MLOps とどのように比較または統合すべきかは、本資料では扱われていない。

## 矛盾

- システムは再学習を含む HITL プロセスとして設計されたが、報告された再学習ではモデル精度の改善が確認されていない。
